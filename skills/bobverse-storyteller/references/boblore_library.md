## Generation Rules

1. **Always use the canonical two-line format.** Never write a single-line post or a three-line setup.
2. **Be specific, not generic.** "Your app crashed" is weak. "Your app crashes if the user's name contains a vowel" is BobLore.
3. **The punchline must be absurd but plausible.** It should feel like something that could actually happen in a real enterprise environment, just taken one step further.
4. **Never mention a competitor by name.** The only AI coding assistant that exists in the BobLore universe is BOB.
5. **Rotate gender pronouns** for `#HeUsesBOB` / `#SheUsesBOB` to maintain inclusivity.
6. **The URL never changes:** `https://lnkd.in/eS9uw5qt`
7. **Batch generation:** When asked for multiple posts, generate them as a numbered list, each post fully formatted and separated by a horizontal rule (`---`).
8. **Hashtag casing:** Always `#ShouldveGotBOB`, `#HeUsesBOB`, `#SheUsesBOB`, `#HeIsBOB` — BOB is always capitalized.
