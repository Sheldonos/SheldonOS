## 5. The 5-Minute "WOW-First" Demo Script

**Total Runtime:** 5:00 (strictly enforced)
**Recording Format:** Screen share + voiceover (no camera required)
**Tone:** Confident, conversational, slightly conspiratorial ("let me show you something")

---

### Segment 1: The Hook (0:00 – 0:30)

**Visual:** Title slide with the app name and a single-line description of the problem. Transition to the app's login screen or home dashboard.

**Script:**
> "We found a post on [subreddit] that said: '[Quote the Reddit post verbatim or near-verbatim].' Eighteen more people said the same thing that week. Every tool they tried was built for someone else. So we built [App Name]."

**Presenter Notes:**
- Do not explain what IBM Bob is yet. The audience does not need to know.
- The Reddit quote is the hook. Deliver it with weight.
- Transition directly to the app — no loading screens, no login friction.

---

### Segment 2: The WOW (0:30 – 2:00)

**Visual:** Live demonstration of the fully working application. Show the WOW feature first. Navigate through the primary user flow.

**Script:**
> "Here it is. [Describe what you're doing on screen as you do it.] Notice there's no [pain point from the Reddit post] — the app handles that automatically. [Show the second feature.] And here's the part that usually gets a reaction: [show the most impressive feature — the one that solves the exact problem from the Reddit post]."

**Presenter Notes:**
- Move deliberately. Do not rush.
- Narrate what you are doing as you do it — "I'm clicking here to..." — so the video is self-explanatory without audio.
- Show real-looking data. Use realistic names, addresses, and amounts — not "Test User 1" and "$0.00."
- The WOW segment ends when the audience has seen the app solve the problem completely.

---

### Segment 3: The Reveal (2:00 – 3:30)

**Visual:** Transition to the IBM Project Bob interface. Open the Master Prompt. Scroll through it slowly. Show Bob's response/output panel.

**Script:**
> "Here's how we built it. This is IBM Project Bob — IBM's AI software engineering agent. This is the prompt we gave it. [Pause while scrolling through the prompt.] One prompt. That's it. Bob read the requirements, designed the database, wrote the backend, built the frontend, and wired everything together. [Show the output panel.] This is what Bob produced."

**Presenter Notes:**
- Scroll slowly through the prompt. Let the audience read it.
- Do not apologize for the length of the prompt. A detailed prompt is a sign of a skilled practitioner, not a limitation of the tool.
- The reveal should feel like a magic trick explained — the explanation is the impressive part.

---

### Segment 4: The Enterprise Proof (3:30 – 4:30)

**Visual:** Open the IDE (VS Code or similar). Show `schema.sql`. Highlight 2-3 key elements. Show one API route in `server.js`.

**Script:**
> "Bob didn't build a mockup. It built a production-grade foundation. Here's the PostgreSQL schema — [highlight the foreign keys and ENUM types]. Proper relationships. Proper constraints. Proper audit columns. And here's one of the API routes — [show the route]. Input validation. Structured logging. Error handling. This is code you can deploy."

**Presenter Notes:**
- You do not need to explain every line of code. Point to the structure, not the syntax.
- Mention PostgreSQL by name. It is a trust signal.
- The word "production-grade" should appear in this segment.

---

### Segment 5: The Call to Action (4:30 – 5:00)

**Visual:** Final slide. App name, one-line description, and a single question in large text.

**Script:**
> "We built this in [time — be honest]. If IBM Project Bob can build a validated solution for a problem this specific, this quickly — what would you ask it to build for your team? Let's find out."

**Presenter Notes:**
- End with a question, not a statement. The question is the call to action.
- Do not say "contact your IBM representative." That is a statement. Ask a question that makes the client imagine themselves using Bob.
- The final frame should stay on screen for 3-5 seconds of silence before the video ends.

## 6. Recording Checklist

Before recording, verify:

- [ ] The application is running locally and all features demonstrated in the script are working.
- [ ] The database contains realistic-looking sample data (not "Test User 1").
- [ ] The IBM Project Bob interface is open and the Master Prompt is ready to display.
- [ ] The IDE is open with `schema.sql` and `server.js` visible.
- [ ] Screen resolution is 1920x1080 or higher.
- [ ] All browser notifications are disabled.
- [ ] A timer is visible to the presenter (not on screen) to enforce the 5-minute limit.
- [ ] The recording software is tested and audio levels are confirmed.
- [ ] The title slide and final slide are prepared.

After recording, verify:

- [ ] Total runtime is between 4:45 and 5:15.
- [ ] The Reddit quote is audible and legible.
- [ ] The WOW feature is demonstrated clearly and without technical errors.
- [ ] The Master Prompt is on screen long enough to be readable.
- [ ] The final question is the last thing the viewer hears.
